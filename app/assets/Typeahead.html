Typeahead

Also known as autocomplete. They can help users choose from a long list.

This is a text input that behaves like a select element, but you can also type in it to filter the options.

Preview

  • Grapes
When autocomplete results are available use up and down arrows to review and enter to select. Touch device users, explore by touch or with swipe gestures.

How to use

1. Download the CSS file

Download accessible-autocomplete.min.css file and add to app/assets/css in this project. If the folder "css" does not exist, create it.

2. Download the JavaScript file

Download accessible-autocomplete.min.js file from app/assets/javascripts in this project.

Download accessible-autocomplete.min.js.map file from app/assets/javascripts in this project.

3. Add this HTML to your HTML page

Add this HTML to the page you want to add the component to:


<link rel="stylesheet" href="/public/css/accessible-autocomplete.min.css">

5. Add JavaScript to the HTML page

Add this JavaScript to the HTML page. You will need to change the fruit-type reference to the id of your select componement.

If you already have a pageScripts block then just add this content to the block.


  {% block pageScripts %}
    <script type="text/javascript" src="/public/javascripts/accessible-autocomplete.min.js"></script>
    <script type="text/javascript">
      var selectEl = document.querySelector('#fruit-type')
      accessibleAutocomplete.enhanceSelectElement({
        autoselect: true,
        confirmOnBlur: true,
        defaultValue: "",
        minLength: 2,
        selectElement: selectEl
      })
    </script>
  {% endblock %}

There are a few different options for enhanceSelectElement. This is a selection:

  • showAllValues - when selected the typeahead shows all possible values (without typing anything)
  • minLength - the number of characters types before the filtering of options happens
  • dropdownArrow - the SVG shown as the dropdown arrow
  • source - use this to change how the filtering works, and the order of results
